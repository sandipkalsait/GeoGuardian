CONTRIBUTING TO GEOGUARDIAN

Thank you for your interest in contributing to GeoGuardian! Your contributions help improve the project, but to protect both you and the project, all contributions must follow the guidelines below.

---

1. Contributor License Agreement (CLA)

Before submitting any code, all contributors must sign the Contributor License Agreement (CLA). This ensures GeoGuardian has the legal rights to use, modify, and distribute your contributions.

- Read & Sign the CLA here: [CLA.md](./CLA.md)
- No pull requests will be merged without a signed CLA.

---

2. How to Contribute

1. Fork the repository and clone it locally:
   https://github.com/sandipkalsait/GeoGuardian.git
2. Create a new branch for your feature or bugfix:
   git checkout -b feature/your-feature-name
3. Make your changes in your branch.
4. Write clear commit messages following the convention:
   [TYPE] Short description of your change
   Example: [FIX] Corrected geofence boundary calculation
5. Run tests to ensure your changes do not break existing functionality.
6. Push your branch to your fork:
   git push origin feature/your-feature-name
7. Open a Pull Request (PR) against the main repository and reference the CLA.

---

3. Coding Standards

- Use null safety for all Dart/Flutter code.
- Follow consistent formatting and indentation.
- Write meaningful comments for complex logic.
- Ensure all code passes static analysis and linting.

---

4. Reporting Issues

If you encounter a bug or have a feature request:

1. Check the existing issues to avoid duplicates.
2. Open a new issue with:
   - Clear description of the problem or feature.
   - Steps to reproduce (if bug).
   - Expected vs. actual behavior (if bug).
   - Screenshots or logs (if applicable).

---

5. Code of Conduct

All contributors are expected to:

- Be respectful and professional.
- Communicate clearly.
- Follow the principles of open collaboration.

Unacceptable behavior will not be tolerated and may result in removal from the project.
